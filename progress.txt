3. SHACL → BRICK IMPORTER
   - Core engine:  brick_app_v2/core/shacl_importer.py
       SHACLImporter.import_file()          single .ttl → library
       SHACLImporter.import_directory()     whole folder, per-file + combined
       SHACLImporter.import_common()        cross-file shared shapes only
       SHACLImporter.import_turtle_string() for API file uploads
       Idempotent: re-running overwrites in-place (same brick_id, same file)
   - CLI wrapper: parse_shacl_to_bricks.py  (thin argparse front-end)
       python parse_shacl_to_bricks.py <dir|file> [options]
       --prefix     library name prefix
       --library    target lib (single-file mode)
       --no-per-file
       --combined   NAME   deduplicated combined lib
       --common     NAME   shapes shared across ≥N files
       --min-files  N      threshold for --common (default 2)
       --repo       PATH   override brick repository path
   - API endpoint: POST /api/import/shacl  (brick_app_v2/api/web_api.py)
       multipart upload: field 'file' (.ttl), optional 'library', 'prefix'
