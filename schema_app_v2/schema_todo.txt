## Architecture Notes

### Multi-Frontend Support
- **NEW**: Multi-tenant backend with session management
- Qt interface uses dedicated qt_session
- Web interface uses dedicated web_session
- Each session has isolated SchemaCore, FlowEngine, BrickIntegration instances
- Shared data through shared_libraries
- Event system for cross-session communication

### State Management
- UIStateManager with SchemaState automaton
- States: INITIAL, SCHEMA_SELECTED, SCHEMA_MODIFIED, SCHEMA_SAVING, COMPONENT_ADDING, FLOW_EDITING
- Validates state transitions
- Controls widget visibility based on state
- Emits signals for state changes

### Backend Architecture
- MultiTenantBackend coordinates sessions
- SessionManager creates and manages sessions
- SchemaBackendSession provides isolated backend per session
- EventRouter routes events to appropriate handlers
- Support for Qt, Web, and API client types

## Event System

### Event Types
- SCHEMA_CREATED, SCHEMA_UPDATED, SCHEMA_LOADED, SCHEMA_SAVED, SCHEMA_DELETED
- COMPONENT_ADDED, COMPONENT_REMOVED
- ROOT_BRICK_SET, FLOW_UPDATED
- LIBRARY_CHANGED
- ERROR_OCCURRED, DIALOG_REQUESTED, STATUS_MESSAGE

### Event Handlers
- QtEventHandler - Emits Qt signals
- WebEventHandler - Sends via WebSocket
- APIEventHandler - HTTP callbacks
- EventRouter - Routes events based on client type
